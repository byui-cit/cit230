Greenville Complete

Overview

Earlier we thought about a new version of the weather app and designed some wireframes and put a bit of thought into styles for our new version. Now it is time to build that page.

Tasks

  1. Create a new file in your editor and name it according to our naming rules. (greenville.html might be appropriate). Add your HTML boilerplate (review the siteplan to HTML activity for a reminder of what I mean by boilerplate) to get your new page started.

  2. Next follow the same process that we did when converting the Franklin wireframe into HTML for your wireframes. Start by adding markup to represent content areas, then fill in the content.

    Just like we did on the earlier activities I would highly recommend making yourself a To-do list or two. Write down with as much detail as you can the steps you will need to follow to build your page.

  3. Just as we did for the first page, add the css to get the layout of a the elements as shown in your wireframe.

    One CSS file or two?

    At this point you may be wondering where to put your CSS for this new page. Normally all the pages in a site would use the same CSS file. In this case though, where we are exploring different versions of the same page it would be more appropriate, and cause fewer problems, if we create a new CSS file for the new layout.

  4. Next add the CSS to style your new page. Change fonts, colors, and spacing to match your plans, and make your page follow the PARC design principles.

  5. Finally just as we did with Franklin make your new page responsive. Flowing naturally from your small wireframe to the large as the browser size changes.
  6. Validate your page for HTML and CSS. Fix all errors.
  7. Make sure the link on the Franklin weather page to this page works, and that you can get back to Franklin from this page. Then make sure to commit and sync your work in Github Desktop. Check your URL (should look like githubusername.github.io) to make sure all of your changes show, and to check that your images all work.

Grading Matrix

This assessment will be graded for four objectives: 1, 2, 3, and 6.

Objective 1 - 18 points

  • The page has a meaningful content-related title and description - 3 points
  • The page is named following the web naming rules - 3 points
  • The page uses semantic structural elements (e.g. header, nav, main, footer) - 3 points
  • All images load correctly - 3 points
  • The page is valid HTML5 - 3 points
  • Navigation items are present for all three cities listed in the weather.json file (Franklin and Greenville nav items should work) - 3 points

Objective 2 - 15 points

  • All CSS is applied with an external stylesheet - 3 points
  • CSS is organized with whitespace and comments - 3 points
  • Primary layout of the page is done with Flexbox with minimal use of position or float - 3 points
  • Responsiveness: The page is responsive and requires no zooming or horizontal scrolling at any size. - 3 points
  • The page is valid CSS3 - 3 points

Objective 3 - 9 points

  1. The page is visually appealing (PARC and typography design principles have been applied). - 3 points
  2. Color scheme and typography indicated in the planning document have been applied - 3 points
  3. All text on the page is readable. (color and size) - 3 points

Objective 6 - 10 points

  1. The assignment is complete as indicated above. - 3 points
  2. Assignment submitted on time - 4 points
  3. No Grammar or spelling errors - 3 points

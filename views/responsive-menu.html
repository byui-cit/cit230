Responsive Navigation Bar "Fix"

Overview

Imagine that you are a teacher, and you need a good example of a responsive navigation bar to show your students. All semester long you have been stressing several 'best practices':

  1. Mobile first design,
  2. Flexbox for layout,
  3. and semantics.

You find the following example of responsive horizontal navbar and you like the simplicity of the solution, but notice it breaks all 3 of the points above. Sigh...it was such a nice example too...but maybe it could be fixed you think! Yes, you think it can.

Tasks

  1. With your learning group, carefully review this responsive horizontal navbar example. Notice how it responds to screen size changes. Seek for understanding of the html and css used to create the menu.
  2. Now create a file in your editor and copy/paste the example into it. 'Fix' the example by addressing the three points above. Your example should be mobile first, use only flexbox for layout, and use semantic elements more appropriately.

  3. Tips

    • Begin by shrinking your screen (or turn on the mobile view in the developer tools in your browser), removing all the floats, and thinking about what steps need to happen to rebuild the menu with flexbox.
    • One of the challenges is the fact that the menu is split...some options on the left, and some on the right. It might be helpful to think of them as a left menu and right menu...the right menu is just really short :)
    • Make any changes to the HTML first that you think you will need, and then start with the CSS. There is not really a footer on this page...but don't forget your header, nav, and main!
    • Remember that if you are working mobile first you will usually want to watch min-width, not max-width
  4. Validate your page. Fix all errors.
  5. Make sure to add a link to your temporary homepage to this new page. Then make sure to commit and sync your work in Github Desktop. Check your URL (should look like githubusername.github.io) to make sure all of your changes show, and to check that your images all work.

Grading Matrix

This assessment will be graded for two objectives: 1 and 2.

Objective 1 - 12 points

  1. The page is valid HTML5 - 3 points
  2. The page has a descriptive title - 3 points
  3. The page makes good use of semantic elements (should have a header, main, and nav) - 3 points
  4. There is a working link back to the Temporary home page - 3 points

Objective 2 - 12 points

  1. Layout of the navbar is done with Flexbox - 3 points
  2. Responsiveness: navbar responds well to screen size changes, adapting as the original did - 3 points
  3. CSS shows a mobile first philosophy (styles outside the media query are for the mobile size, adaptations in the media query are for the large screen size) - 3 points
  4. The page is valid CSS3 - 3 points

This week we need to apply the rest of the styling to polish things up and add CSS code to make our page responsive: Adaptable to changes in the size of the screen it is viewed on.

Responsiveness

Make sure to have completed the Introduction to Responsive Web Design course on sitepoint before proceeding with this activity.

At the core of the responsive design movement...the structure responsible for bring the magic as it were, is the @media query. A query is a question or condition, and so a "media query" is a question or condition about the media our page is currently being displayed on. The browser can track this, and we can make our designs adapt as a result.

  4. Lets add some details. Refer to the style guide and find the portion on typography. Notice that fonts were specified for both headlines and body text. Lets change those now.

    Font is governed by the font-family property. Notice in the example on that site that they have 3 fonts listed. This is for redundancy in case the browser cannot find the first choice. Fonts are listed in order of: preferred font, second choice, generic font family.

    If we want to change the font for all of our text where should we do it?

    If you go back and look at that link for font-family again scroll down just a bit and you will find a table with information about the property...the one we are looking for starts with "Default value". Right after that you see "Inherited: yes". Some CSS properties only apply to the element to which they are applied. Other elements (like font-family) will be applied to the element to which they are attached, and all of it's descendents. So if we want all of our text to use a certain font, we could apply it at the body, and since everything is inside that element, everything will inherit that font!

    Make a rule for body applying the list of body fonts to it in your css.

    Make another rule in your css that will apply the headline font to all headlines.

    The style guide only gave us one font for our headlines! We need to find one more font that is somewhat similar to the desired font to add to our list. The best would be if you can find one of the web safe fonts to use as our second choice. Then choose the best fit from the generic font families for the 3rd choice: cursive, fantasy, mono-space, serif, sans-serif

    One other problem, what should we use for the selector if we want to change all headlines? whenever you have the same or similar styles you want applied to more than one selector, you can list the styles once, and use commas to add the selectors...like this:

    
h1,
h2,
h3,
h4,
h5,
h6 {
    font-family: Sansita, "second choice", generic-family;
}

    After adding the css above (and replacing the placeholder text with your choices) you might have noticed that the headline fonts did not change! Sansita is not a font that is found on most computers, and computers can only display fonts they know. If we want to use Sansita, we have to provide it. I found the Sansita font on Google Fonts, a collection of royalty free fonts that Google makes available. If you go there and search for "sansita" it will pull up the font. Click on the plus, then on the black box that appears.

    Google lists two ways to add the font: Standard, and @Import. I prefer the @import method...click on that tab and it will give you some CSS to add to your file.

    Important! Add that @import line to the very top of your css! It should look something like this:

    
@import url('https://fonts.googleapis.com/css?family=Sansita');

  5. Let's adjust fonts sizes now. As I look at the page in the browser, and especially if I look at it on an actual phone all the font feels a bit small. I think an increase across the board would be a good thing. The default font size for browsers is 16px. Lets try upping that to 20px. Add the following line to your body rule in your css.

    
font-size: 20px;

    Did you notice when you added that last line that all of the fonts increased? Including the headlines? If you inspect element an look at one of the headline in the developer tools you will notice a section of css that is being applied that you didn't write. The styles are on a light grey background and labeled "user agent stylesheet"

    These are the default browser styles and on an h1 you will notice font-size: 2em; If you refer back to the Sitepoint article on units that you were asked to read this week it will remind you that "em" is a relative unit. As the base or parent font size gets bigger...so will everything using an "em" to describe it's size.

    Before we added that line 1em = 16px, after that line 1em = 20px. So our H1 at 2em would now be 40px instead of 32px.

    Use the font-size property to adjust the size of all of the text on the page to match the small wireframe. Adjusting the base size got us close, but there are adjustments that need to be made. You may need to add some classes to a few elements to accomplish this.

  6. Next we need to adjust some background colors. The header, footer, and nav should all have colored backgrounds as specified in our list. I will leave this to you. The property you will need to apply to those 3 elements is background-color

  7. Links are next. The style guide specified that our links should be styled in a particular way, and that they should be interactive...change when the use hovers over them. How do we do this?

    We have learned how to target elements...things that exist in our page, but to do this task we need to target the state of an element. Enter pseudo-classes. These are special selector that do just what we need here...they target things that don't really exist in the DOM (all of the objects or elements on our page). One of the pseudo-classes is hover. That resource actually has a good example of styling all the states of a link...exactly what we need to do. flexbox.

    So to style my navigation links normal state I would do something like this:

    
nav a:link {
    text-decoration: none;
    color: #47556F;        
}

    First notice the selector I used. I used the descendent selector to target only links that are in a nav element. I really don't want these styles to apply to all links. Then I changed 2 things: removed the underline (text-decoration) and changed the color of the text.

    Add the CSS to style the other states of the links: visited, hover, active.

    Two things to watch for on this. CSS is often indifferent about spaces between things. It will treat color: #47556F and color:#47556F exactly the same. It is not always indifferent however.

    nav a:link is not treated the same as nav a: link. You can't have any spaces between the ":" on pseudo-class selectors.

    The other issue is that the order that we define the states matters. For them to all work right they should be defined in the css in this order: link, visited, hover, active

    As always if you have any problems with your CSS, Validate!

  8. The Wireframe shows that we should have an image behind most of the content. You may have notcied that I did not have you add that image earlier when we were adding content. In this case it would be better to add the image as a background image. Lets do that now.

    Background images are actually pretty easy to use. We need to decide which element should host the image. In this case it looks like pretty much all of the content in main is on the image...so lets attach it there. We need an image that is the right size and shape. We will talk about working with images more later, but for now you can use this image. Download the image to your web directory and then add css similar to this to your main rule (make sure to correct the path based on where you put the image):

    
background-image: url("images/sunny-winter.jpg");
background-size: cover;

    Review the link above to understand what both of these properties are doing and what other options you have with using them.

  9. The styleguide show a very specific styling for the content boxes on the page. It included this image:

    It also included the styles you will need to make something look like that: 

    
background-color: rgba(200,200,200,.6);
margin: .2em auto;
border: 1px solid;
border-radius: 5px;

    Lets look at each of those lines.

    The first we have used before, background-color, you may not have seen that way of specifying color however. RGB functions similarly to hexidecimal, except that the values range from 0-250. (250,250,250) would be white. Interesting thing about colors, whether hex or rgb...if you have the same value for all 3 parts...you get grey. High numbers give a light grey, low numbers give a dark grey.

    You may have also noticed that this is a rgba value. The a stands for alpha...it's another way to say transparency. The ".6" value there means make this color 60% visible.

    The next line is to give a bit of separation between content boxes that follow each other. The auto for the second value also works to center the element horizontally.

    The next 2 lines place a border around the element, and make the corners rounded.

    Using the CSS above apply the styles to the location, temperature box, and current conditions box. You may need to add some classes to do this.

  10. Lets skip to item k. Rotating an element is done with the transform property. Spend some time on that page to get an idea of what you can do with transforms. For now we will focus on rotate. Try applying the following css to one of the dice icons. 

    
.rotate25 {
    transform: rotate(25deg);
    display: inline-block;
}

    I think 25deg is quite jaunty! If you disagree then feel free to change the angle of rotation until you find something you like.

Web Usability

Usability "is a quality attribute that assesses how easy user interfaces are to use. The word "usability" also refers to methods for improving ease-of-use during the design process." (Usability 101: Introduction to Usability).

The study of usability is extensive and complex. This week we will only introduce the concept of usability and suggest some ways that you can increase the usability of your web pages.

Semantics

Already in the course we have discussed and used semantic markup. The use of the HTML5 structural elements (header, nav, main, aside, footer, section and article) themselves add to the usability of a web page. Back in week 3 you were to read about HTML5 Semantics. Included was a link to a blog post about content organization using semantics and how these elements have inherent meaning to both computers and humans. You did read the blog post, right?

Usability Checklist

The WAVE website has a great introduction page to usability. Included is a list of "Principles of Accessible Design". This list contains 12 items that are useful for enhancing usability.

Take a Look

With the other members of your learning team, open and read through the Principles of Accessible Design. Talk amoung yourselves to make sure you all understand each of the 12 items. If not, click the link associated with the item to learn more.

The Golden Rule

Remember the golden rule -- Do unto others as you would have done unto you! It applies to web sites too. If you come to a site, you expect to be able to find content quickly and easily, read it without difficulty (despite the size of the screen on your device), view pictures and other media without having to scroll constantly or be greeted with messages telling you to download and install some plugin to see and use the content, be able to find your way around the site and the like. All of this may seem like common sense, and it is, but it is also usability.

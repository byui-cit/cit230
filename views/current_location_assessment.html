Current Location Page

Work With a Partner or Group

You are required to work with a partner or group (maximum of 5) for this assessment. Each person is responsible for her or his own work, but you should collaborate to assist one another to complete the assessment.

List Who You Worked With

In the index page of the folder you will download below you will find a comment between the "head" and "body" elements for you to list who you worked with. List all names in the comment, with your name first. These should be members of our class!

Be Aware!

Unlike previous assessments, this one will require it to be done twice. Once using the seed code files provided below and again in the weather pages you began working with back in week 5.

Get the Seed Code

Below is a link for you to download seed code to use for this assessment.

jsonweather-html.zip

  1. Once downloaded, unzip the file and move the "jsonweather-html" folder to the root of your local web site.
  2. Open the folder and become familiar with the files that are included.

Get The Data

Most of the code for this assessment will be written in the remote_wu.js file. However, two other small pieces of code will need to be written. First, you will need to hook up the remote_wu.js file to the index page (see line 45 of the index page). Second, you may need to add custom CSS as explained in step 8.3 below!

  1. You will probably want to review the Weather Underground links provided in the Weather Underground API activity as they indicate How to request data and the Type of data that you will need.
  2. Use the getJSON() method which begins on line 25 and currently ends on line 35.
  3. Write all code within this method, add new lines within the method as need.
  4. Send the URL to the weather underground API requesting both the geolocation and conditions objects. Use the method that allows you to send in the latitude and longitude for the current location. Don't forget to include your API key as required in the URL.
    1. The latitude (lat) and longitude (long) were gotten for you on lines 10 and 11.
  5. Write a success function and store its parameter as "data".
  6. Use console.log often to see what you are getting back from the API and your own variables.
  7. Base Data:
    1. You are to get the city and state and combine them into a single location name separated by a comma and space. The name is to be displayed in the h1 element with an id of cityDisplay AND as the first item in the "title" element within the "head" element of the page (do not replace what's there, add this in front of what's already there).
    2. You are to get the current temperature and display it into the li with an id of currentTemp. Make sure you use JavaScript to round the number to the closest integer and display the degree symbol following the number.
    3. Finally, you are to display a summary of the current conditions in the li with an id of summary. Be sure the text starts with a capital letter (Hint: use the weather property from the current_observation object).
  8. Additional Data:
    1. You are to add a minimum of two additional data points to the weather page.
      1. Three list items are provided in the index page, each with an id of "add1", "add2" and "add3" for your use.
      2. Be sure that whatever you add makes sense from the site visitors perspective, don't make them guess what you meant - make it clear and obvious!
      3. You may alter the CSS within the user_styles.css file to style this content if needed (this file is already set to be used, you don't need to do anything with the main stylesheet).

Thoroughly Test

Make sure you thoroughly test the page on your local system to insure that all data works and is clearly visible as you intend it to be on all screen sizes. Test using the Chrome developer tools and device emulators.

Add the Collaboration List

Add the names of the people that you collaborated with to the comment in the index page (between the head and body elements prior to uploading.

Create a Link

On your temporary home page, create a link to the seed code index page within the jsonweather-html folder.

Upload the altered temporary home page and the entire jsonweather-html folder.

Remote Testing

Thoroughly test the links and operation of your code on the production server.

Note: As of version 50, the Chrome browser does not support the geoLocation javascript function unless the domain is a secured domain using the https protocol! Most shared hosting sites do not use https. The easiest solution is to test your page using a different browser (FireFox or Safari) instead of using Chrome.

However, our GitHub repositories do use the https protocol so we should not have a problem.

Grading Matrix

Objective 5 (30 points)

Two links appear on the temporary home page: 1) to the seed code index page and 2) to the weather site index page. Both pages work to show the current location weather. - 5 points.

Using the weather site index page, the data is successfully returned and displayed in the web page using:

  • The getJSON method found in an external JavaScript file.  - 5 points
  • The three base data points are present:
         The location name is the city and state, separated with a comma and space - 3 points
         The location name appears in the content "h1" and as the first value in the "title" element - 2 points
         The temperature includes the degree symbol - 2 point
  • The summary value starts with a capital letter - 2 point
  • A minimum of two additional data points are included. - 6 points
  • The additional data points are clearly legible and communicate the information unambiguously. - 5 points

Objective 6 (5 points)

The weather site index page comment lists the other class members that the developer collaborated with during the assessment.

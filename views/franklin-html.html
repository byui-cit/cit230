Getting started - Franklin

Overview

Last week we were introduced to the Planning Document for our weather site and we gained some experience working with HTML. We also had a chance to practice converting wireframes into HTML. Now we are ready to begin building our weather site by creating the Franklin page by following the wireframe in the planning document.

Which Wireframe?

As we are getting ready to start you may be wondering which of the two wireframes should we follow? The answer is both! Since we are concentrating on getting the content off of the wireframe and into HTML we need to make sure we get all of the content. Some content will be hidden on either the large or small screen, so if we just look at one or the other we might miss something.

Tasks

  1. Review the small and large wireframes to get an idea of what we need to do. Keep them handy as we will be referring back them often.
  2. Create a new html document in your editor and save it in the weather folder with a name that keeps in mind the naming rules we talked about earlier for the Franklin page of our site.

  3. Start your page by adding all of the critical parts of a valid HTML5 webpage: doctype, html, head, metas, title, body, etc. Or you can use the code below as a starting point.

    
<!doctype html>
<html lang="en-us">
    <head>
         <meta charset="UTF-8">
          <title>My first website</title>
          <meta name="description" content="Short Description of the page.">
          <meta name="viewport" content="width=device-width">
    </head>
    <body>
        <h1>Hello World!</h1>
    </body>
</html>

    If you are unsure what any of those lines do...make sure to revisit your notes or the readings from this unit. W3schools is a great resource as well. You might consider moving through the first few pages of the reference linked above if parts of the code above are unclear.

  4. Let's continue by changing the title and description to something that makes more sense for your page. Both of these elements are actually really important for Search Engine Optimization (SEO). This is the ability of search engines to figure out what your page is about.

    Ideally your title should contain the name of the current page, the name of the site, and some keywords that describe what the site is about. Here is a short video on Titles that explains more.

    The description should do just what it says...describe what the site...and particularly this page are about.

  5. Now we are ready to start adding our content. but first we need to create spaces in our HTML for that content to go. HTML is considered a Semantic language. Basically the html markup use should describe the content it contains whenever possible.

    Lets add a few semantic elements to get us started. Most documents have the same underlying parts. Most of us learned those when learning to write our first 5 paragraph essay. They are: header, main body, and footer.

    HTML5 provided us with elements to represent those parts. Lets add the following to our html. this should go in the body, replacing the h1 that is currently there (remember that the body contains all the stuff we want to be in the browser window.

    
    <header>
        <h1>Mobile Weather App Site Plan</h1>
        <nav><a href="">Home</a> </nav>
    </header>
    <main>

    </main>
    <footer>
        &copy;2017 Your name here
    </footer>

    You may be saying to yourself at this point "this sounds very familiar..." and you would be right. The last few steps are exactly what we did last week when creating the HTML version of the planning document. This is not an accident, turns out that there is a certain amount of repetition in web development. The core of an HTML document is really very similar from site to site.

    Because of this many developers will create what is know as 'boilerplate' code. It is a simple version of an HTML project that has all of the parts that are almost always used. There is even an addon in Brackets that will provide you with these templates. The boilerplate that I tend to use looks like this:

    
<!doctype html>
<html lang="en-us">
    <head>
         <meta charset="UTF-8">
          <title>Page | Website | Keywords</title>
          <meta name="description" content="Short Description of the page.">
          <meta name="viewport" content="width=device-width">
          <link rel="stylesheet" type="text/css" href="main.css" >
    </head>
    <body>
        <header>
            <nav>
                <ul>
                    <li><a href="#">Home</a></li>
                </ul>
            </nav>    
        </header>
        <main>

        </main>
        <footer>
            &copy;2017 Your name here
        </footer>
    </body>
</html>

    Most of that should look familiar. There are two lines we have not discussed yet in the head. The link element and the viewport meta. We will learn about those next week. Feel free to use my boilerplate, or come up with your own with the things that you find you always want in an HTML page.

  6. From here we are ready to continue adding content areas. Everything else should go in the header, main, or footer elements we added. As you add content pay attention to what type of information it is and try to use the appropriate HTML element.

    Make sure to refer often to the wireframes to make sure you don't miss anything.

    I would recommend starting with outlining content areas...start general and work towards specifics. For example if we look at the large wireframe you will notice that we have a good start on the general content areas. But if we look specifically at the main content we see that we need to carve out more defined spaces for things. We need 3 more containers inside of <main> in fact...one for current conditions, one for the location and temperature, and one for hourly information. You may want to refer to the sectioning flowchart to help you decide which elements you should use to define those areas.

    After you define those 3 areas, look specifically at the center box. It seems to be divided into a couple more areas...one for the location, and then another for the temp and high and low. Add elements to contain those bits of content.

  7. Once you have all of the content areas defined go back and add the actual content to the correct area. Again at this point you should be using links, images, lists, and lots of paragraphs and headlines.
  8. Validate your page. The easiest way to do this would be to use the Developer tools you installed in the first week. You should see a little gear icon on the top right side of your browser if they are installed. Click on the gear, then Tools and if you are working locally (URL looks like: 127.0.0.1) then pick the "Validate Local HTML" option. If you have already uploaded your html to Github and want to validate there (URL looks like username.github.io) then pick the Validate HTML option. Fix all errors.
  9. Make sure to add a link to your temporary homepage to this new page. Then make sure to commit and sync your work in Github Desktop. Check your URL (should look like githubusername.github.io) to make sure all of your changes show, and to check that your images all work.

Grading Matrix

This assessment will be graded for two objectives: 1 and 6.

Objective 1 - 18 points

  • The page has a meaningful content-related title and description - 3 points
  • The page is named following the web naming rules - 3 points
  • The page uses semantic structural elements (e.g. header, nav, main, footer) - 3 points
  • All images load correctly - 3 points
  • The page is valid HTML5 - 3 points
  • Navigation items are present for all three cities listed in the weather.json file (nav items do not need to work) - 3 points

Objective 6 - 10 points

  1. All of the content from the wireframes is present. - 3 points
  2. Assignment submitted on time - 4 points
  3. No Grammar or spelling errors - 3 points

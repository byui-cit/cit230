Project Proposal

The scope and functionality of the project will be determined by you and described in a project proposal. The finished project must demonstrate your mastery of building a responsive, valid html and css website that makes appropriate use of JavaScript and/or JQuery along with data from JSON file(s). Usability concepts to meet the needs of users and design principles to enhance the appearance of the web site are also expected.

The final proposal must thoroughly describe the project to be built, including:

  1. The purpose of the project from a content delivery point of view.
  2. The target audience and persona for the project;
  3. A list and description of the web pages to be built (minimum of 5 content pages).
  4. Two wireframe sketches of a typical content page for small and large screens. Be sure to add detail to the sketches to clearly indicate the type of content (text, images, video, etc...) and layout on various devices.
  5. A light Style Guide including a minimum of a Typography section and colors section.
  6. Be a valid HTML document.
  7. Be submitted to the Project Proposal dropbox during week 07.

You should continue with the web page produced in part one of this assignment and add the additional sections outlined above.

Professor response will be in one of the following forms:

  1. Approved – you may proceed with the project as proposed.
  2. Modify – you must make modifications to the project proposal as indicated and resubmit the modified proposal to the professor.
  3. Denied – the project does not meet the requirements or is too large or small of scope. You must submit a new proposal.

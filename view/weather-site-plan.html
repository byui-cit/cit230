Weather Site Introduction

Overview

Beginning today we will be working on a website to provide weather information for three cities. You will be provided the planning document describing the site purpose, target audience, a persona to represent a member of the audience and wireframes showing how the data will be laid out on the screen for two device sizes (small and large screens). Your first task is to read over the document carefully so that you understand what will be expected for this project. The next task will be to convert the document to HTML and post it to your site.

Tasks

  1. Download the weather site planning document and carefully read through it.
  2. Create a folder in your web root called 'weather'. Then create a new html document in your editor and save it in that folder with a name that keeps in mind the naming rules we talked about earlier
  3. Create an "images" folder inside of the "weather" folder you just created. Then download the document images and place them insde that folder. (that is a .zip archive of images. Remember to expand it! Most of the time placing it in the correct folder and then double-clicking it will do the trick.)

  4. Start your page by adding all of the critical parts of a valid HTML5 webpage: doctype, html, head, metas, title, body, etc. Or you can use the code below as a starting point.

    
<!doctype html>
<html lang="en-us">
    <head>
         <meta charset="UTF-8">
          <title>My first website</title>
          <meta name="description" content="Short Description of the page.">
          <meta name="viewport" content="width=device-width">
    </head>
    <body>
        <h1>Hello World!</h1>
    </body>
</html>

    If you are unsure what any of those lines do...make sure to revisit your notes or the readings from this week. W3schools is a great resource as well. You might consider moving through the first few pages of the reference linked above if parts of the code above are unclear.

  5. Let's continue by changing the title and description to something that makes more sense for your page. Both of these elements are actually really important for Search Engine Optimization (SEO). This is the ability of search engines to figure out what your page is about.

    Ideally your title should contain the name of the current page, the name of the site, and some keywords that describe what the site is about. Here is a short video on Titles that explains more.

    The description should do just what it says...describe what the site...and particularly this page are about.

  6. Now we are ready to start adding our content. but first we need to create spaces in our HTML for that content to go. HTML is considered a Semantic language. Basically the html markup use should describe the content it contains whenever possible.

    Lets add a few semantic elements to get us started. Most documents have the same underlying parts. Most of us learned those when learning to write our first 5 paragraph essay. They are: header, main body, and footer.

    HTML5 provided us with elements to represent those parts. Lets add the following to our html. this should go in the body, replacing the h1 that is currently there (remember that the body contains all the stuff we want to be in the browser window.

    
    <header>
        <h1>Mobile Weather App Site Plan</h1>
        <nav><a href="">Home</a> </nav>
    </header>
    <main>

    </main>
    <footer>
        &copy;2017 Your name here
    </footer>

    Notice I went ahead and added a nav element. That wasn't part of the essay we learned to write...but navigation is a pretty important part of the web. The header is often where it goes.

    You might have also noticed the footer. There is a funny code there. It is called an html entity code and can be recognized by the leading &. We use them to display special characters that are could not normally be displayed. When you put that into your HTML it will get converted into a ©

  7. From here we are ready to start adding content. Everything else should go in that main element we added. As you add content pay attention to what type of information it is and try to use the appropriate HTML element.

    For example if we look at the second section we see a headline "Audience". Audience is a section of the document under the main title right? We used an h1 for the title of the document...so it would make sense to use the first subheading tag (h2) for the Audience title. Audience also has sub-sections: "Audience Goals", and "Personas". Since they are sub-headings of an h2 identified section, they should probably use the h3 element to represent them.

    The "Content" heading is not a part of the "Audience" section and so it should go back to an h2 element to display it on the page.

  8. Add the remainder of the content in the document to your HTML. Use the semantic element that best describes the content. You should use links, images, lists, probably a simple table for the colors, and lots of paragraphs and headlines.
  9. Make sure to add a link to your temporary homepage to this new page. Then make sure to commit and sync your work in Github Desktop. Check your URL (should look like githubusername.github.io) to make sure all of your changes show, and to check that your images all work. then post your url to Ilearn.

Grading Matrix

This assessment will be graded for two objectives: 1 and 6.

Objective 1 - 20 points

  1. The page is valid HTML5 - 3 points
  2. The page has a descriptive title and description - 4 points
  3. The page has semantic header, nav, main and footer elements - 3 points
  4. All images load correctly - 4 points
  5. There is a working link back to the Temporary home page - 3 points
  6. The page has a meaningful title and each section has an appropriate heading - 3 points

Objective 6 - 8 points

  1. All of the content from the provided document is in the web page - 5 points
  2. Assignment submitted on time - 3 points
